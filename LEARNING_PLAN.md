# 20 周路线图

进度由能力验收决定。周数是建议节奏，不是必须追赶的截止日期。

| 周次 | 学习主题 | 主项目产出 |
| --- | --- | --- |
| 1 | 终端、文件、Python、Git 入门 | 第一个命令行程序与学习仓库 |
| 2 | 变量、字符串、数字、布尔值 | 文档信息展示程序 |
| 3 | 条件、循环、列表、字典、集合 | 文档列表与筛选功能 |
| 4 | 函数、文件、JSON、异常 | 本地文档读取与搜索 |
| 5 | 模块、类、类型标注、异步概念、pytest | 可测试的命令行文档管理器 |
| 6 | HTTP、JSON、REST、FastAPI | 第一个文档 API |
| 7 | Pydantic、校验、异常、日志 | 稳定的文档增删查接口 |
| 8 | SQL、SQLite、ORM、接口测试 | 持久化文档服务 |
| 9 | LLM、Token、上下文、提示词 | 可替换的模型调用接口 |
| 10 | Embedding、切分、向量检索 | 文档索引与检索管线 |
| 11 | 手写 RAG、引用、拒答 | 带来源的知识问答 |
| 12 | 评测集、召回率、幻觉分析 | 40 道问题的评测基线 |
| 13 | Agent、状态、工具、决策循环 | 三个知识库工具 |
| 14 | LangGraph、节点、路由、记忆 | 可追踪的 Agent 工作流 |
| 15 | 流式输出、重试、超时、降级 | 完整对话链路 |
| 16 | PostgreSQL、pgvector、迁移 | 生产型数据存储 |
| 17 | Streamlit、配置、安全与限流 | 可操作的网页产品 |
| 18 | Docker、CI、部署、综合测试 | 在线演示版本 |
| 19 | README、架构图、指标、演示视频 | 完整作品集 |
| 20 | 简历、项目讲解、面试训练 | 求职材料与模拟面试 |

## 阶段验收

每个阶段都必须同时满足：

1. 能用自己的话解释核心概念。
2. 能独立完成一道变式练习。
3. 能读懂并修改自己提交的代码。
4. 自动化测试通过。
5. 阶段测验达到 80%。

## 最终项目边界

第一版包含：

- TXT、Markdown、PDF 文档上传与索引
- 文档管理、对话、引用和用户反馈 API
- 检索、读取片段和列出文档三个 Agent 工具
- 对话记忆、流式输出、重试与拒答
- FastAPI、PostgreSQL、pgvector、LangGraph、Streamlit
- pytest、Docker Compose、GitHub Actions 和在线部署

第一版不包含多 Agent、模型微调、复杂权限系统或 Kubernetes。
