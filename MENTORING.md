# 我们的师生协作方式

## 每节课的结构

1. 用 5 分钟回忆上节课。
2. 用生活化例子理解一个新概念。
3. 一起完成一个最小示例。
4. 你独立完成一道变式练习。
5. 我们分析错误并更新学习记录。

## 你遇到问题时

请把下面三样内容直接发给我：

1. 你执行的命令或修改的代码。
2. 完整报错文字，不要只发最后一行。
3. 你认为问题可能在哪里。

我会按“现象 → 原因 → 排查方法 → 修复”的顺序讲解，并优先给提示，不直接夺走你的思考机会。

## 代码帮助等级

- 一级：给你一个问题，引导你自己发现。
- 二级：指出相关概念或代码范围。
- 三级：给伪代码或半成品。
- 四级：给完整示例，并要求你完成变式练习。

默认从一级提示开始。你可以随时说“我卡住了，请升一级提示”。

## 每周固定产物

- 一项可运行的功能
- 至少一个测试
- 一篇学习日志
- 一条错误档案
- 十张英文术语卡
- 一次有意义的 Git 提交
